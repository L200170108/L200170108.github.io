DDL

DDL - Data Definition Language

merupakan kumpulan perintah SQL yang digunakan untuk membuat, mengubah dan menghapus struktur dan definisi metadata dari objek-objek database. Seperti definisi yang telah dijelaskan di atas, DDL adalah kumpulan perintah SQL yang digunakan untuk membuat (create), mengubah (alter) dan menghapus (drop) struktur dan definisi tipe data dari objek-objek database.

Objek-objek database pada yang dimaksud - pada MySQL - adalah sebagai berikut :
1. Database
2. Table
3. View
4. Index
5. Procedure (Stored Procedure)
6. Function
7. Trigger

Contoh :
Perintah berikut akan membuat table pada database kita dengan nama "ms_karyawan"
CREATE TABLE ms_karyawan (
kode_cabang varchar(10) default NULL,
kode_karyawan varchar(10) NOT NULL,
nama_depan varchar(8) default NULL,
nama_belakang varchar(9) default NULL,
jenis_kelamin varchar(1) default NULL,
PRIMARY KEY (kode_karyawan) )

Berikut adalah contoh perintah DDL yang digunakan pada MySQL:
A. Pembuatan (CREATE)
1. CREATE DATABASE
2. CREATE FUNCTION
3. CREATE INDEX
4. CREATE PROCEDURE
5. CREATE TABLE
6. CREATE TRIGGER
7. CREATE VIEW

B. Perubahan (ALTER & RENAME)
1. ALTER DATABASE
2. ALTER FUNCTION
3. ALTER PROCEDURE
4. ALTER TABLE
5. ALTER VIEW
6. RENAME TABLE

C. Penghapusan (DROP)
1. DROP DATABASE
2. DROP FUNCTION
3. DROP INDEX
4. DROP PROCEDURE
5. DROP TABLE
6. DROP TRIGGER
7, DROP VIEW
